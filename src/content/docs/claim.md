# Получить наследство

Страница **«Получить наследство»** предназначена для наследников: по файлу heir\_\*.json можно увидеть, что положено, и забрать долю после истечения периода неактивности наследодателя.

## Что нужно

- Файл **heir\_\<id\>\_\<имя\>.json**, выданный наследодателем. В нём должны быть поля: **owner**, heirId, leaf, proof и либо **sharesBps** (открытые доли), либо **encryptedShares** + **shamirShare** (зашифрованные доли). При зашифрованных долях в файле также есть **shamirThreshold** (сколько долей нужно для восстановления) и **shamirTotal** (сколько всего долей).
- Подключённый кошелёк (адрес наследника).

Адрес хранилища (vault) определяется автоматически по **owner** через фабрику: `factory.vaults(owner)`. При необходимости адрес vault можно ввести вручную.

## Расшифровка долей (если файл зашифрован)

Если в файле наследника доли зашифрованы (поле `encryptedShares`), перед получением наследства нужно восстановить ключ шифрования:

1. На странице отображается: **«Для восстановления ключа нужно X из Y долей»** (X = shamirThreshold, Y = shamirTotal).
2. Ваша доля ключа из файла подставляется автоматически.
3. Введите недостающие доли — от других наследников (если ключ был разделён между наследниками) или от доверенной организации (если ключ был разделён на 2 доли).
4. Нажмите **«Расшифровать доли»**. После успешной расшифровки откроется стандартная информация о хранилище и таблица «Вам положено».

### Извлечь только долю ключа из файла

На этой же странице есть блок **«Извлечь долю ключа из файла»**:

- можно загрузить файл наследника `heir_<id>_<имя>.json` или файл организации `organization_key_share.json`;
- приложение локально (в вашем браузере) читает поле `shamirShare` и показывает только строку доли ключа (hex), которую можно скопировать и переслать другим участникам;
- сам JSON-файл никуда не отправляется и не сохраняется.

## После загрузки файла

Отображаются:

- **Наследодатель** — адрес owner из файла.
- **Хранилище** — адрес vault (подставляется из фабрики или вводится вручную).
- **Последняя активность** — дата и время последнего подтверждения активности наследодателем.
- **До конца периода** — обратный отсчёт до момента, когда можно забрать токены («осталось X дн. Y ч. Z мин.» или «период истёк»).
- **Можно забрать токены** — да / нет (с пояснением: период не истёк, DMS не включён или доля уже получена).

## Таблица «Вам положено по токенам хранилища»

Строится по списку токенов из контракта vault. Для каждого токена показываются:

- **Токен** — символ и название (например, ETH, USDC (USD Coin)).
- **Доля %** — ваша доля в процентах из файла (sharesBps).
- **Баланс хранилища** — текущий баланс vault по этому токену.
- **Вам положено** — расчёт: баланс хранилища × (доля % / 100).

## Получение доли

Когда период истёк, DMS включён и вы ещё не забирали эту долю:

1. Нажмите **«Получить наследство»**.
2. Подтвердите транзакцию в кошельке (вызов `claimWithProof` на контракте vault).

Транзакция переведёт вашу долю по каждому токену (включая ETH) на ваш кошелёк. Повторно ту же долю (тот же leaf) забрать нельзя.
