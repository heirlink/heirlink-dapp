# Распределение наследства

Форма позволяет создать Merkle tree для распределения токенов и нативного ETH между наследниками и записать root и список токенов в хранилище на контракте.

## Токены

1. Отметьте галочку **ETH**, если нужно включить нативный Ethereum в расчёт долей (в контракт передаётся адрес-заполнитель для ETH; на страницах управления и получения наследства ETH отображается отдельно с нативным балансом).
2. Нажмите **«+ Добавить токен»** и введите адреса ERC-20 токенов (формат `0x...`).
3. Балансы отображаются автоматически при подключённом кошельке; числа выводятся с разделителями тысяч (например, 100 000 000).

## Режим «Доли в файлах»

Над списком токенов есть переключатель **«Скрыть доли» / «Раскрыть доли»**.

- **Раскрыть доли** (по умолчанию) — в файлах наследников (heir\_\*.json) доли по токенам хранятся открыто (sharesBps).
- **Скрыть доли** — доли шифруются AES-256-GCM. Генерируется случайный ключ, которым шифруется массив sharesBps; ключ затем разделяется по схеме Шамира. Наследник не может узнать свою долю и вызвать контракт без восстановления ключа.

### Два способа разделения ключа

1. **Разделить ключ между наследниками** — ключ делится на N частей (по числу наследников), каждый наследник получает в своём файле уникальную долю ключа. Порог восстановления вы задаёте вручную (от 2 до N долей). Наследники сами ответственны за координацию и восстановление ключа.
2. **Разделить ключ на 2 доли (с организацией)** — все наследники получают одинаковую первую долю ключа, вторая доля скачивается в отдельный файл (**organization\_key\_share.json**). Этот файл передаётся доверенной организации, которая выйдет на связь с наследниками и предоставит свою долю для расшифровки.

В обоих случаях файл **merkleTree.json** (для наследодателя) содержит полные данные — шифруются только файлы наследников.

## Наследники

1. Введите **ID наследника** — целое число (1, 2, 3...)
2. Укажите **имя** (опционально)
3. Заполните **доли в %** по каждому токену
4. Сумма долей по каждому токену не должна превышать 100%

## Результат

После нажатия "Обновить хранилище" или "Создать новое хранилище":

- Скачиваются файлы: **merkleTree.json** (root, токены, массив листьев, полный массив наследников) и по файлу на каждого наследника (**heir\_\<id\>\_\<имя\>.json**). В режиме «Раскрыть доли» файл содержит **sharesBps** открыто; в режиме «Скрыть доли» — **encryptedShares** (зашифрованные доли) и **shamirShare** (доля ключа). Общие поля: owner, heirId, name, leaf, proof. При режиме «с организацией» дополнительно скачивается **organization\_key\_share.json**.
- Инициируется **транзакция**: создаётся новое хранилище (если его ещё нет) или обновляется существующее — в контракт передаются root и список токенов (включая адрес для ETH, если галочка ETH отмечена). Подтвердите транзакцию в кошельке.
- Поле **«Период неактивности (дней)»** используется только при первом создании хранилища (при следующей генерации вызывается уже обновление существующего vault).
