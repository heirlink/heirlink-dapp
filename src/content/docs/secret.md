# Тесты (страница «Секрет»)

На этой странице собраны вспомогательные инструменты: разделение секрета по Шамиру и проверка Merkle proof.

---

## Секретное разделение (Shamir)

Разделение секрета по схеме Шамира (Shamir's Secret Sharing). Не связано с блокчейном; всё выполняется в браузере.

### Как это работает

Секрет разбивается на несколько частей (shares). Для восстановления нужна только часть из них (порог).

### Параметры

- **Порог (threshold)** — минимальное количество частей для восстановления
- **Части (shares)** — общее количество частей, на которые делится секрет

### Пример

При пороге 2 и 3 частях: секрет можно восстановить, имея любые 2 из 3 частей.

## Проверка Merkle proof

Проверка, что лист наследника входит в дерево с заданным корнем (без вызова контракта).

1. **Файл дерева** — загрузите **merkleTree.json** (объект с полем `root` и опционально `tokens`).
2. **Файл с пруфом** — загрузите файл наследника **heir\_\*.json** (объект с полями `leaf`, `proof` и опционально `owner`).
3. Нажмите **«Проверить proof по root»**. Отобразится результат: пруф верен (лист входит в дерево с данным корнем) или неверен.

Формат leaf и proof совпадает с тем, что ожидает контракт vault при вызове claimWithProof.
