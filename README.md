# Heirlink dApp

dApp для сохранности цифрового капитала и передачи его наследникам по смарт-контракту (Dead Man's Switch). Владелец задаёт токены и доли наследников; при длительной неактивности наследники получают средства по Merkle-пруфам.

## Возможности

- **Наследство** — ввод токенов (ERC-20 и ETH) и наследников с долями, генерация Merkle tree, скачивание merkleTree.json и heir\_\*.json, создание/обновление хранилища на контракте.
- **Управлять наследством** — просмотр vault, подтверждение активности (DMS), включение/отмена DMS, изменение периода, балансы, вывод токенов и ETH, таблица долей по загруженному файлу (с символами и именами токенов).
- **Получить наследство** — для наследников: загрузка heir\_\*.json, информация о наследодателе и хранилище, последняя активность и оставшееся время до конца периода, таблица «вам положено» по токенам, получение доли (claimWithProof).
- **Документация** — обзор, распределение наследства, управление, получение наследства, тесты (Shamir, проверка Merkle proof).

## Стек

- Next.js, React, TypeScript
- wagmi, viem, RainbowKit
- Merkle tree (merkletreejs), формат долей в bps

## Конфигурация

- Сеть и адреса контрактов — в `src/contracts/addresses.json` (chainId, factory). Подключите кошелёк к указанной сети.

## Запуск

```bash
npm install
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000). Документация — в разделе «Документация» в навигации.
